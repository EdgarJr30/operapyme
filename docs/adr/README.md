# ADRs

Usar esta carpeta para decisiones tecnicas o de producto que:

- cambian la direccion del sistema
- introducen tradeoffs fuertes
- afectan seguridad, datos o experiencia transversal

## Formato recomendado

1. Contexto
2. Decision
3. Consecuencias
4. Alternativas descartadas

## Primeros ADRs sugeridos

- Supabase como backend principal con RLS obligatoria
- auditoria total por tablas y eventos
- stress tool como herramienta separada
