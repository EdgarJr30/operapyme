# Reglas de UI institucional y publica

Este documento aplica a superficies no operativas del SaaS, como futuras landings, paginas informativas o experiencias publicas.

## Reglas

- deben explicar el valor del producto rapido
- no competir visualmente con el backoffice
- mantener consistencia con el branding general
- evitar claims tecnicos imposibles de verificar
- no exponer datos internos ni rutas administrativas

## Estado

La superficie publica esta reservada, no es prioridad de fase 1.
