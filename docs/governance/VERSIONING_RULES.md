# Reglas de versionado

## Commits

- hacer commits frecuentes por bloques coherentes
- no mezclar cambios sin relacion en un mismo commit cuando se pueda evitar
- mensajes cortos y especificos

## Pull requests

- una PR debe explicar impacto de producto, arquitectura y testing
- si toca seguridad o datos, debe explicarlo explicitamente

## Release discipline

- no liberar cambios de permisos o migraciones sin pruebas asociadas
- documentar breaking changes antes de fusionar
