# Reglas de documentacion

## Principios

- documentar en la misma tarea
- espanol first dentro de `docs/`
- mantener el repo como fuente de verdad formal

## Cuando actualizar docs

- cambia arquitectura
- cambia dominio
- cambia setup
- cambia seguridad o permisos
- cambia UX visible
- cambia testing o scripts

## Regla de consistencia

- no dejar referencias rotas
- no hablar del producto heredado
- no afirmar como implementado algo que solo esta reservado
